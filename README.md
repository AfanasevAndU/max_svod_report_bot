# max_svod_report_bot

Бot для MAX, который следит за сдачей отчетов в 1С:Свод отчетов. Данные о статусах отчетов берутся из ClickHouse, и по расписанию в чат MAX отправляются напоминания о том, что нужно сдать сегодня, что нужно сдать до конца дня и что уже просрочено.

## Как это работает

1. Бот забирает из ClickHouse все отчеты, у которых `status != 'Подготовлен'`.
2. Для каждого отчета вычисляется точный дедлайн (дата + время) на основе поля `report_period` и комментария `comment` (см. ниже).
3. Отчеты фильтруются:
   - отчеты вне текущих суток не рассматриваются вообще (ни просрочка "за неделю", ни "завтрашние" не отправляются — только сегодняшний день);
   - ежедневные отчеты без указанного времени сдачи в комментарии не отправляются в MAX;
   - отчеты с меткой-исключением в комментарии (например "Другой отдел") не отправляются в MAX независимо от срока.
4. По расписанию (`node-cron`, часовой пояс из `TIMEZONE`) бот шлет сообщения в чат MAX:
   - **09:00** — все отчеты, которые нужно сдать сегодня;
   - **14:00** и **17:00** — отдельным сообщением отчеты, которые еще нужно сдать до конца дня, и отдельным сообщением уже просроченные (дедлайн сегодня, но время уже прошло).
5. Если текст сообщения превышает лимит MAX API (4000 символов), он автоматически разбивается на несколько сообщений с пометкой «(часть N из M)».

## Формат `report_period`

Поддерживаются три формата, от которых зависит, как вычисляется базовая дата дедлайна:

| Формат | Пример | Тип отчета | Базовая дата |
|---|---|---|---|
| Диапазон дат | `01.07.2026 - 09.07.2026` | недельный | конец диапазона |
| Конкретный день | `9 июля 2026 г.` | ежедневный | указанный день |
| Месяц | `Июль 2026 г.` | ежемесячный | последний день месяца (если не переопределено комментарием) |

## Формат `comment`

Комментарий задает точное время сдачи, а для ежемесячных отчетов — еще и точный день:

```
Время сдачи: 10:00
```
```
Дата сдачи: 09.07; Время сдачи: 10:00
```

- `Время сдачи: HH:MM` — время дедлайна, комбинируется с датой из `report_period`.
- `Дата сдачи: DD.MM` — переопределяет день/месяц дедлайна (год берется из `report_period`); используется для ежемесячных отчетов, где сам период указывает только месяц.

### Метки-исключения

Если комментарий содержит одну из меток из списка `EXCLUDED_COMMENT_MARKERS` в [`src/periodParser.js`](src/periodParser.js), отчет не отправляется в MAX вообще, независимо от дедлайна. Сейчас в списке: `Другой отдел`. Чтобы добавить свою метку — допишите строку в этот массив.

## Установка

```bash
npm install
```

Создайте `.env` в корне проекта со следующими переменными:

| Переменная | Обязательна | По умолчанию | Описание |
|---|---|---|---|
| `MAX_BOT_TOKEN` | да | — | токен бота MAX |
| `MAX_CHAT_ID` | да | — | ID чата, куда отправлять сообщения |
| `CLICKHOUSE_HOST` | да | — | адрес ClickHouse (со схемой, например `http://host`) |
| `CLICKHOUSE_PORT` | нет | `8123` | порт ClickHouse |
| `CLICKHOUSE_USER` | нет | `default` | пользователь ClickHouse |
| `CLICKHOUSE_PASSWORD` | нет | `""` | пароль ClickHouse |
| `CLICKHOUSE_DATABASE` | да | — | база данных |
| `CLICKHOUSE_TABLE` | да | — | таблица с отчетами (`report_period`, `organization`, `report_name`, `status`, `comment`) |
| `TIMEZONE` | нет | `Asia/Yekaterinburg` | часовой пояс для cron-расписания |
| `NODE_ENV` | нет | `development` | `development` включает подробные цветные логи (pino-pretty) |

## Запуск

```bash
npm start        # обычный запуск
npm run dev       # запуск с автоперезапуском при изменении файлов (node --watch)
```

Если ClickHouse или MAX API требуют самоподписанный сертификат, потребуется переменная окружения `NODE_TLS_REJECT_UNAUTHORIZED=0`:

```bash
# PowerShell
$env:NODE_TLS_REJECT_UNAUTHORIZED = "0"; npm start
```

## Структура проекта

```
src/
  app.js           точка входа, запускает планировщик
  scheduler.js      cron-расписание (9:00, 14:00, 17:00) и логика рассылки
  checker.js        загрузка отчетов и их классификация (сегодня / позже сегодня / просрочено)
  clickhouse.js     клиент ClickHouse и запрос отчетов
  periodParser.js   разбор report_period/comment, вычисление дедлайна, метки-исключения
  formatter.js      формирование текста сообщений для MAX (с разбивкой на части)
  bot.js            отправка сообщений через MAX Bot API
  config.js         валидация переменных окружения (zod)
  logger.js         логирование (pino)
```
