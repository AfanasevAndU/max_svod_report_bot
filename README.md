# max_svod_report_bot

Бot для MAX, который следит за сдачей отчетов в 1С:Свод отчетов. Данные о статусах отчетов берутся из ClickHouse (таблицу заполняет загрузчик [`svod_reports`](../svod_reports)), и по расписанию в чат MAX отправляются напоминания о том, что нужно сдать сегодня, что нужно сдать до конца дня и что уже просрочено.

## Как это работает

Таблицу `svod_reports` наполняет отдельный загрузчик из JSON-API 1С. Он уже отбирает
только нужные отчёты — с флагом `bot = true`, непустым `deadline_time` и `period_end = сегодня` —
и хранит снимок за каждый день (`load_date`). Поэтому боту остаётся простая работа:

1. Бот забирает из ClickHouse сегодняшний снимок (`load_date = сегодня`), у которого `state != 'Подготовлен'`.
2. Для каждого отчёта дедлайн собирается напрямую: дата `period_end` (= сегодня) + время `deadline_time` (`HH:MM`).
3. По расписанию (`node-cron`, часовой пояс из `TIMEZONE`) бот шлёт сообщения в чат MAX:
   - **утро (09:00)** — все отчёты, которые нужно сдать сегодня;
   - **день (14:00)** и **вечер (17:00)** — отдельным сообщением отчёты, которые ещё нужно сдать до конца дня, и отдельным сообщением уже просроченные (время дедлайна уже прошло).
4. Если текст сообщения превышает лимит MAX API (4000 символов), он автоматически разбивается на несколько сообщений с пометкой «(часть N из M)».

> Отбор отчётов (какие вообще попадают боту, недельные/месячные, метки «другой отдел») теперь
> выполняется на стороне загрузчика через флаг `bot` в API 1С — разбирать форматы периода и
> комментарии в этом проекте больше не требуется.

## Таблица `svod_reports`

Бот читает столбцы, которые пишет загрузчик:

| Столбец | Назначение |
|---|---|
| `organization` | Медицинская организация |
| `report_type` | Название отчёта |
| `state` | Статус (напоминания шлём при `state != 'Подготовлен'`) |
| `period` | Период отчёта (для отображения/логов) |
| `period_end` | Дата окончания периода = дата дедлайна |
| `deadline_time` | Время сдачи `HH:MM` |
| `template_comment` | Комментарий из шаблона |
| `load_date` | Дата загрузки (снимок за день) |

## Установка

```bash
npm install
```

Создайте `.env` в корне проекта со следующими переменными:

| Переменная | Обязательна | По умолчанию | Описание |
|---|---|---|---|
| `MAX_BOT_TOKEN` | да | — | токен бота MAX |
| `MAX_CHAT_ID` | да | — | ID чата, куда отправлять сообщения |
| `CLICKHOUSE_HOST` | да | — | адрес ClickHouse (со схемой, например `http://host`) |
| `CLICKHOUSE_PORT` | нет | `8123` | порт ClickHouse |
| `CLICKHOUSE_USER` | нет | `default` | пользователь ClickHouse |
| `CLICKHOUSE_PASSWORD` | нет | `""` | пароль ClickHouse |
| `CLICKHOUSE_DATABASE` | да | — | база данных |
| `CLICKHOUSE_TABLE` | да | — | таблица, которую пишет загрузчик `svod_reports` (см. столбцы выше) |
| `TIMEZONE` | нет | `Asia/Yekaterinburg` | часовой пояс для cron-расписания |
| `MORNING_CRON` | нет | `0 9 * * *` | расписание утренней рассылки («сдать сегодня») |
| `MIDDAY_CRON` | нет | `0 14 * * *` | расписание дневной рассылки («ещё сдать» + «просрочено») |
| `EVENING_CRON` | нет | `0 17 * * *` | расписание вечерней рассылки («ещё сдать» + «просрочено») |
| `NODE_ENV` | нет | `development` | `development` включает подробные цветные логи (pino-pretty) |

## Запуск

```bash
npm start        # обычный запуск
npm run dev       # запуск с автоперезапуском при изменении файлов (node --watch)
```

Если ClickHouse или MAX API требуют самоподписанный сертификат, потребуется переменная окружения `NODE_TLS_REJECT_UNAUTHORIZED=0`:

```bash
# PowerShell
$env:NODE_TLS_REJECT_UNAUTHORIZED = "0"; npm start
```

## Структура проекта

```
src/
  app.js           точка входа, запускает планировщик
  scheduler.js      cron-расписание (утро/день/вечер) и логика рассылки
  checker.js        загрузка отчетов и их классификация (сегодня / позже сегодня / просрочено)
  clickhouse.js     клиент ClickHouse и запрос сегодняшнего снимка отчётов
  periodParser.js   сборка дедлайна из period_end + deadline_time
  formatter.js      формирование текста сообщений для MAX (с разбивкой на части)
  bot.js            отправка сообщений через MAX Bot API
  config.js         валидация переменных окружения (zod)
  logger.js         логирование (pino)
```
